# Инструкция по развертыванию приложения «Ячейка»

Данный проект настроен для автоматического деплоя на виртуальный сервер (VPS) с автоматической выдачей SSL-сертификатов (HTTPS) через веб-сервер Caddy и CI/CD пайплайн в GitHub Actions.

---

## Шаг 1. Подготовка VPS сервера и Домена

1. **Арендуйте сервер (VPS/VDS)** с операционной системой **Ubuntu 22.04 LTS** или **Ubuntu 24.04 LTS**.
   * *Рекомендуемые минимальные характеристики:* 2 CPU, 4 GB RAM (для стабильной сборки и работы Docker).
2. **Зарегистрируйте домен** (например, `yacheyka.ru`).
3. В панели управления вашим доменом добавьте **A-запись**, указывающую на IP-адрес вашего VPS.
   * *Пример:* Имя: `@` (или оставить пустым), Тип: `A`, Значение: `123.45.67.89` (IP вашего сервера).

---

## Шаг 2. Первичная настройка на VPS

Подключитесь к вашему VPS по SSH и выполните команды для обновления пакетов и установки Docker:

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Docker и Docker Compose
sudo apt install -y docker.io docker-compose git

# Добавление Docker в автозапуск
sudo systemctl enable --now docker
```

---

## Шаг 3. Настройка доступа для GitHub Actions

Для того чтобы GitHub Actions мог подключаться к вашему серверу по SSH и обновлять код:

1. **Сгенерируйте SSH-ключи** на вашем локальном компьютере (или на VPS):
   ```bash
   ssh-keygen -t rsa -b 4096 -f ~/.ssh/yacheyka_deploy_key
   ```
2. **Добавьте публичный ключ** на сервер VPS:
   * Скопируйте содержимое файла `yacheyka_deploy_key.pub` (публичный ключ).
   * Откройте на сервере файл `~/.ssh/authorized_keys` и вставьте ключ новой строкой.
   * Убедитесь, что права доступа верны:
     ```bash
     chmod 700 ~/.ssh
     chmod 600 ~/.ssh/authorized_keys
     ```

---

## Шаг 4. Настройка секретов в репозитории GitHub

Перейдите в ваш репозиторий на GitHub в раздел **Settings** -> **Secrets and variables** -> **Actions** -> **New repository secret** и добавьте следующие секреты:

### Системные секреты (для SSH-подключения):
* **`SSH_HOST`** — IP-адрес вашего сервера (например, `123.45.67.89`).
* **`SSH_USERNAME`** — имя пользователя для подключения (обычно `root`).
* **`SSH_PRIVATE_KEY`** — содержимое вашего приватного SSH-ключа (скопируйте весь текст из файла `yacheyka_deploy_key`, включая строки `-----BEGIN RSA PRIVATE KEY-----` и `-----END RSA PRIVATE KEY-----`).

### Продакшн секреты приложения (для генерации `.env` файла):
* **`DB_USERNAME`** — имя пользователя базы данных (например, `yacheyka_admin`).
* **`DB_PASSWORD`** — пароль базы данных (придумайте сложный пароль).
* **`DB_DATABASE`** — имя базы данных (например, `yacheyka`).
* **`MINIO_ACCESS_KEY`** — логин от панели MinIO/S3 (минимум 3 символа, например, `minio_root`).
* **`MINIO_SECRET_KEY`** — пароль от панели MinIO/S3 (минимум 8 символов, придумайте сложный).
* **`JWT_SECRET`** — секретный ключ для шифрования токенов сессий (придумайте любую длинную случайную строку).
* **`TELEGRAM_BOT_TOKEN`** — токен вашего Telegram-бота от BotFather.
* **`TELEGRAM_BOT_USERNAME`** — имя пользователя вашего бота без символа `@` (например, `yacheyka_bot`).
* **`DOMAIN_NAME`** — доменное имя вашего сайта (например, `yacheyka.ru` или `subdomain.yacheyka.ru`).

---

## Шаг 5. Запуск деплоя

Теперь при каждом `git push` в ветку `main` или `feat-telegram-auth` GitHub Actions автоматически запустит процесс развертывания:
1. Зайдет по SSH на сервер VPS.
2. Склонирует проект в `/app/yacheyka` (если это первый запуск) или обновит его (`git pull`).
3. Сгенерирует продакшн-файл `.env` из ваших GitHub Secrets.
4. Выполнит сборку и перезапуск контейнеров через `docker-compose.prod.yml`.
5. Caddy автоматически закажет SSL-сертификаты, и сайт станет доступен по HTTPS на вашем домене!

---

## Как проверить статус развертывания на VPS

Вы можете подключиться к серверу и выполнить следующие команды для отладки:

```bash
# Перейти в папку проекта
cd /app/yacheyka

# Посмотреть статус запущенных контейнеров
docker-compose -f docker-compose.prod.yml ps

# Посмотреть логи бэкенда
docker-compose -f docker-compose.prod.yml logs -f backend

# Посмотреть логи Caddy (проверить статус выпуска SSL)
docker-compose -f docker-compose.prod.yml logs -f caddy
```
