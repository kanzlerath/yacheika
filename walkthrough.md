# Результаты доработки проекта «Ячейка»

Мы успешно адаптировали интерфейс PWA-приложения для мобильных устройств (учёт безопасных зон iOS Safe Areas), открыли публичный доступ для гостей без принудительной авторизации, создали панель настроек (смена стилей карты, GPS-сортировка по близости) и внедрили красивую защиту интерактивных действий.

---

## 1. Адаптация под безопасные зоны (iOS Safe Areas)

В режиме PWA без границ (fullscreen/standalone с `apple-mobile-web-app-status-bar-style: black-translucent`) элементы интерфейса ранее перекрывались системными индикаторами iOS.

Мы реализовали адаптивный макет с использованием переменных CSS `env(safe-area-inset-*)`:
* **Глобальный макет**: Контейнер зафиксирован во весь экран (`h-screen h-[100dvh] overflow-hidden`). Контентный блок занимает всё свободное пространство (`flex-1 min-h-0`).
* **Шапка приложения (Header)**: Добавлен отступ `padding-top: env(safe-area-inset-top, 0px)` и динамическая высота `calc(4rem + env(safe-area-inset-top, 0px))` для компенсации высоты статус-бара.
* **Боковая панель (DiscoveryPanel)**: Получила нижний отступ `pb-[calc(1.5rem+env(safe-area-inset-bottom,0px))]`, чтобы список не уходил под системный индикатор жестов iOS.
* **Карточка заведения (VenueCard)**: Высота компактного вида адаптируется к `calc(256px + env(safe-area-inset-bottom, 0px))`, нижние отступы кнопок также выровнены.
* **Плавающие кнопки**: Кнопка «Открыть подборки» приподнимается над зоной свайпа: `bottom: calc(1.5rem + env(safe-area-inset-bottom, 0px))`.

## 2. Гостевой доступ и настройки (Новое)

1. **Доступ без авторизации**:
   * При первом входе приложение не блокирует экран. Карта Новосибирска, списки заведений, события и подборки загружаются сразу в гостевом режиме.
2. **Панель настроек (SettingsModal)**:
   * Вызывается по кнопке **«Войти / Настройки»** (или по аватарке профиля, если вошли) в правом верхнем углу шапки.
   * Позволяет на лету менять стиль подложки карты, включать сортировку по GPS и управлять авторизацией.
3. **Смена тем карты**:
   * В настройках доступны три стиля: **Тёмная** (Dark Matter), **Цветная** (Voyager) и **Светлая** (Positron). Смена темы происходит мгновенно на уровне MapLibre без пересоздания HTML-маркеров заведений.
4. **Сортировка по близости (GPS)**:
   * Тумблер в настройках запрашивает права на геопозицию. 
   * При согласии на карте рендерится **синяя пульсирующая точка** вашего местоположения, а список заведений пересортировывается в порядке удаленности (ближайшие заведения выводятся в топ через PostGIS-запрос с расчетом расстояний).
5. **Защита гостевых реакций (AuthPromptModal)**:
   * Если гость нажимает кнопки «Хочу пойти» / «Не моё место» или пытается проголосовать за вайб дня в заведении, всплывает окно-предупреждение с предложением войти в один клик через официальный Telegram Login Widget.

## Проверка и запуск

Все конфигурации в Docker Compose обновлены. Для тестирования локально или через HTTPS-туннель:

### 1. Запуск контейнеров
```bash
docker compose up --build -d
```

### 2. Тестирование сценариев:

#### Сценарий А. Гостевой вход и смена тем
1. Очистите кэш `localStorage` (или откройте инкогнито) и перейдите на [http://localhost:3002](http://localhost:3002).
2. **Ожидаемо**: Приложение открывается сразу. Видна тёмная карта Новосибирска и список заведений.
3. Нажмите в шапке кнопку **«Войти / Настройки»**.
4. В блоке «Стиль карты» выберите **Цветная** или **Светлая**.
5. **Ожидаемо**: Подложка карты мгновенно меняется, маркеры заведений сохраняют координаты. Выбранная тема сохраняется при обновлении страницы.

#### Сценарий Б. Геолокация и GPS-сортировка
1. Откройте панель настроек в шапке.
2. Включите тумблер **«Сортировка по близости (GPS)»**.
3. Браузер запросит доступ к местоположению — разрешите его.
4. **Ожидаемо**:
   * Карта плавно переместится (flyTo) к вашему текущему местоположению.
   * На карте появится синяя точка с мягкой анимацией-пульсацией.
   * Список заведений слева отсортируется по удаленности от вас.

#### Сценарий В. Защита реакций гостя
1. Будучи неавторизованным, откройте карточку любого заведения (например, Friends Bar).
2. Нажмите кнопку **«Хочу пойти»** или проголосуйте за любой тег во вкладке «Народный вайб».
3. **Ожидаемо**: Появится всплывающее окно **«Войдите через Telegram»** с виджетом для входа, объясняющее необходимость авторизации для этого действия.
4. Нажмите «Продолжить просмотр гостем» для закрытия окна без авторизации.

#### Сценарий Г. Вход и права администратора
1. Откройте окно настроек или вызовите окно защиты реакций.
2. Войдите через Telegram-виджет аккаунтом `@nick_luzhkov` (ID `1859857121`).
3. **Ожидаемо**:
   * Авторизация проходит успешно, профиль отображается в настройках.
   * В шапке вместо кнопки настроек появляется ваша аватарка с юзернеймом.
   * В панели настроек появляется пункт **«Панель Управления (CRUD)»** для переключения в админку.
   * Все лайки и вайб-теги теперь прожимаются и сохраняются в БД.

## 3. Обновление UI панели управления (Админки)

Мы адаптировали панель администрирования для комфортной работы со смартфонов, исправили визуальные недочеты и добавили удобный выбор гео-координат.

1. **Мобильный Master-Detail макет**:
   * На экранах телефонов панель больше не вытягивается в одну бесконечную страницу. Внедрен двухэкранный режим:
     * **Экран списка** (Master): Отображает перечень заведений и живые логи аналитики.
     * **Экран редактора** (Detail): Открывается при выборе заведения или создании нового. Сверху добавлена кнопка **«← Назад к списку заведений»** для переключения.
2. **Маска ввода для номера телефона**:
   * Поле «Телефон» теперь оснащено динамическим маскированием. При вводе цифр они автоматически преобразуются в единый формат `+7 (XXX) XXX-XX-XX`.
3. **Устранение бага сдвига тумблера Premium**:
   * Контейнеру переключателя добавлен класс `relative`, что надежно зафиксировало круглый ползунок внутри трека тумблера, убрав его перекос.
4. **Выбор координат на карте с телефона**:
   * В мобильной анкете заведения добавлена кнопка **«Указать на карте»**.
   * При клике открывается полноэкранный интерактивный режим карты, на котором можно установить булавку простым тапом.
   * Выбрав точку, администратор нажимает **«Подтвердить точку»**, возвращается к анкете, где видит аккуратный блок **«Точка выбрана на карте!»** с кнопкой мгновенного применения координат к полям формы.
